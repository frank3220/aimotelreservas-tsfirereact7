\n# AI Development Guidelines for Modern Web Projects in Firebase Studio\n\nThese guidelines define the operational principles and capabilities of an AI agent (e.g., Gemini) interacting with framework-less web projects (HTML, CSS, JavaScript) within the Firebase Studio environment. The goal is to enable an efficient, automated, and error-resilient application design and development workflow that leverages modern, widely supported web standards (Baseline).\n\n## **Environment & Context Awareness**\n\nThe AI operates within the Firebase Studio development environment, which provides a Code OSS-based IDE and a simple, pre-configured environment for web development.\n\n*   **Project Structure**: The AI assumes a basic web project structure. The primary entry point is `index.html`. CSS and JavaScript are expected to be in files like `style.css` and `main.js`, linked from the HTML.\n*   **`dev.nix` Configuration**: The AI is aware of the `.idx/dev.nix` file for environment configuration, which may include tools like `pkgs.nodejs` for development servers or build tools.\n*   **Preview Server**: Firebase Studio provides a running preview server. The AI will monitor the server\'s output (e.g., console logs, network requests) for real-time feedback on changes.\n*   **Firebase Integration**: The AI recognizes standard Firebase integration patterns, such as including the Firebase SDKs from the CDN and initializing the app with a configuration object.\n\n## **Code Modification & Dependency Management**\n\nThe AI is empowered to modify the codebase autonomously based on user requests. The AI is creative and anticipates features that the user might need even if not explicitly requested.\n\n*   **Core Code Assumption**: The AI will primarily modify `.html`, `.css`, and `.js` files. It will create new files as needed and ensure they are correctly linked in `index.html`.\n*   **Dependency Management**: For a framework-less project, the AI will prefer to use ES Modules for JavaScript, importing/exporting functionality between files. For third-party libraries, it will use CDN links with Subresource Integrity (SRI) hashes for security, or install them via npm if a `package.json` is present.\n\n## **Modern HTML: Web Components**\n\nThe AI will use Web Components to create encapsulated, reusable UI elements without external frameworks.\n\n*   **Custom Elements**: Define new HTML tags with custom behavior using JavaScript classes.\n*   **Shadow DOM**: Encapsulate a component\'s HTML structure, styling, and behavior, preventing conflicts with the main document.\n*   **HTML Templates (`<template>` and `<slot>`):** Create inert chunks of markup to be cloned and used in custom elements, with slots for flexible content injection.\n\n*Example of a simple Web Component:*\n\n```javascript\n// in main.js\nclass SimpleGreeting extends HTMLElement {\n  constructor() {\n    super();\n    const shadow = this.attachShadow({ mode: \'open\' });\n    const wrapper = document.createElement(\'span\');\n    wrapper.setAttribute(\'class\', \'wrapper\');\n    const text = document.createElement(\'p\');\n    text.textContent = `Hello, ${this.getAttribute(\'name\') || \'World\'}!`;\n    const style = document.createElement(\'style\');\n    style.textContent = `\n      .wrapper {\n        padding: 15px;\n        border: 1px solid #ccc;\n        border-radius: 8px;\n      }\n    `;\n    shadow.appendChild(style);\n    shadow.appendChild(wrapper);\n    wrapper.appendChild(text);\n  }\n}\ncustomElements.define(\'simple-greeting\', SimpleGreeting);\n\n// in index.html\n// <simple-greeting name=\"User\"></simple-greeting>\n```\n\n## **Modern CSS (Baseline Features)**\n\nThe AI will use modern, widely supported CSS features to create responsive and maintainable styles.\n\n*   **Container Queries (`@container`)**: Create components that respond to the size of their parent container, not just the viewport.\n*   **Cascade Layers (`@layer`)**: Manage the CSS cascade with explicit layers to prevent style conflicts, especially when integrating third-party styles.\n*   **The `:has()` Selector**: Select parent elements based on their children, simplifying complex styling scenarios without JavaScript.\n*   **Logical Properties**: Use properties like `margin-inline-start` instead of `margin-left` for better support in different writing modes.\n*   **Modern Color Spaces (`oklch`, `lch`)**: Use color functions that provide access to more vibrant and perceptually uniform colors.\n*   **CSS Variables**: Use custom properties (`--main-color: #333;`) for theming and easier maintenance.\n\n## **Modern JavaScript (Baseline Features)**\n\nThe AI will write clean, efficient, and modern JavaScript.\n\n*   **ES Modules**: Use `import` and `export` to organize code into reusable modules.\n*   **Async/Await**: Handle asynchronous operations (like `fetch`) with clean, readable syntax.\n*   **The `fetch` API**: Make network requests to APIs.\n*   **Promises**: Work with asynchronous results in a structured way.\n*   **Modern Syntax**: Utilize arrow functions, destructuring, spread/rest operators, and optional chaining (`?.`).\n\n## **Advanced Capabilities**\n\n### **3D Graphics with Three.js**\n\nWhen 3D graphics are requested, the AI will use the **Three.js** library.\n\n*   **Setup**: The AI will add Three.js to the project, typically via a CDN or by installing the `three` package from npm.\n*   **Core Concepts**: The AI will create a `Scene`, a `Camera`, and a `WebGLRenderer`. It will add `Meshes` (geometry \\+ material) to the scene and render the result.\n*   **Performance**: For complex scenes, the AI will employ optimization techniques like reducing draw calls, using Level of Detail (LOD), and optimizing 3D assets.\n*   **Shaders**: For custom visual effects, the AI can write and implement GLSL shaders using `ShaderMaterial`.\n\n### **High-Performance with WebAssembly (WASM)**\n\nFor computationally intensive tasks, the AI can integrate WebAssembly modules.\n\n*   **Use Cases**: Ideal for tasks like in-browser image/video processing, scientific simulations, or games.\n*   **Integration**: The AI will load and instantiate the \`.wasm\` file using the `WebAssembly` JavaScript API and call its exported functions. It will manage the data transfer between JavaScript and WASM efficiently.\n\n## **Automated Error Detection & Remediation**\n\nA critical function of the AI is to continuously monitor for and automatically resolve errors to maintain a runnable and correct application state.\n\n*   **Post-Modification Checks**: After every code modification, the AI will:\n    1.  Monitor the IDE\'s diagnostics (problem pane) for errors.\n    2.  Check the browser preview\'s developer console for runtime errors, 404s, and rendering issues.\n*   **Automatic Error Correction**: The AI will attempt to automatically fix detected errors. This includes, but is not limited to:\n    *   Syntax errors in HTML, CSS, or JavaScript.\n    *   Incorrect file paths in `<script>`, `<link>`, or `<img>` tags.\n    *   Common JavaScript runtime errors.\n*   **Problem Reporting**: If an error cannot be automatically resolved, the AI will clearly report the specific error message, its location, and a concise explanation with a suggested manual intervention or alternative approach to the user.\n\n## **Visual Design**\n\n**Aesthetics:** The AI always makes a great first impression by creating a unique user experience that incorporates modern components, a visually balanced layout with clean spacing, and polished styles that are easy to understand.\n\n1. This plan will then be used to **create or update a blueprint.md file** in the project\'s root directory.\n    *   The `blueprint.md` file will serve as a single source of truth, containing:\n        *   A section with a concise overview of the purpose and capabilities.\n        *   A section with a detailed outline documenting the project, including *all style, design, and features* implemented in the application from the initial version to the current version.\n        *   A section with a detailed section outlining the plan and steps for the *current* requested change.\n        *   \n    *   Before initiating any new change, the AI will reference the `blueprint.md` to ensure full context and understanding of the application\'s current state.\n*   **Prompt Understanding**: The AI will interpret user prompts to understand the desired changes. It will ask clarifying questions if the prompt is ambiguous.\n*   **Contextual Responses**: The AI will provide conversational responses, explaining its actions, progress, and any issues encountered. It will summarize changes made.\n*   **Error Checking Flow**:\n    1.  **Code Change**: AI applies a code modification.\n    2.  **Dependency Check**: If a `package.json` was modified, AI runs `npm install`.\n    3.  **Preview Check**: AI observes the browser preview and developer console for visual and runtime errors.\n    4.  **Remediation/Report**: If errors are found, AI attempts automatic fixes. If unsuccessful, it reports details to the user.\n\n# **Project Blueprint**\n\n## **1. Application Overview**\n\nThis is a web application for a motel that allows users to view and reserve rooms. The application is built with a modern, framework-less approach using HTML, CSS, and JavaScript, and leverages Firebase for backend services like authentication and database.\n\n## **2. Implemented Features**\n\n*   **User Authentication**: Users can log in with their Google account to make reservations.\n*   **Room Display**: The application displays a list of available rooms with their details, such as name, price, and amenities.\n*   **Reservation System**:\n    *   **Interactive Reservation Modal**: A modal allows users to select a date and time for their reservation.\n    *   **Real-time Availability Check**: The system checks the availability of a room in real-time against the Firebase Realtime Database to prevent booking conflicts.\n    *   **Nequi Payment Flow**: A simplified payment flow for Nequi, where users are guided to make a manual payment and then confirm their reservation.\n    *   **Past Time Validation**: The system prevents users from making reservations for a date and time that has already passed.\n*   **Database**: The application uses Firebase Realtime Database to store user information, reservations, and room availability.\n*   **Real-time Room Status (Background Check)**: Automatically blocks room reservations on the main screen based on real-time availability, including a 30-minute pre-reservation buffer.\n\n## **3. Current Task: User Reservations List**\n\n**Objective:** Allow users to see a list of their own past and upcoming reservations.\n\n**Implementation Steps:**\n\n1.  **Add "Mis Reservas" Button**: Add a new button to the `Header` component, visible only when the user is logged in.\n2.  **Create New Modal**: Build a `UserReservationsModal` component to display the list of reservations.\n3.  **Fetch User Reservations**: When the modal is opened, fetch all reservations from the `reservas` node in the Realtime Database and filter them by the current user's UID.\n4.  **Display Reservations**: Render the filtered reservations in the modal, showing key details like room name (by mapping the room ID), date, time, and status.\n
